Pencil Production Line Simulation

Advanced Programming Project

Student: Mostafa Elsayed | Student ID: 100006129

Project Overview

This project simulates a pencil manufacturing production line using Python, a Tkinter-based HMI, InfluxDB, Grafana, and Docker. The system models the assembly process of a pencil and displays production data in a live dashboard.

Production Line Stages

The simulated pencil production line contains five main stages:

  • S1: Graphite core insertion
  • S2: Wooden body assembly
  • S3: Eraser attachment
  • S4: Eraser holder attachment
  • S5: Final quality control

System Architecture

The backend logic is implemented in Python. The HMI allows the user to start, stop, reset, and acknowledge faults. Production data is sent to InfluxDB and visualized in Grafana.

  • Backend: Python production line logic and defect handling
  • Frontend: Tkinter HMI with production state display
  • Database: InfluxDB time-series database
  • Dashboard: Grafana panels for live production monitoring
  • Deployment: Docker Compose for InfluxDB and Grafana containers

Quality Control

The program marks pencils as defective when faults are detected, such as a missing or broken graphite core, cracked wooden body, missing eraser, loose eraser holder, or final quality issue. Defective products are counted and the defect reason is displayed on the HMI.

Grafana Dashboard

The Grafana dashboard displays live production parameters from InfluxDB, including:

  • Produced pencils
  • Defective pencils
  • Machine temperature
  • Machine state
